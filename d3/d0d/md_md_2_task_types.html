To indicate remaining tasks we use some code indicators:

// TODO:
// FIXME:
// BUG:
// WARN:
// HACK:
// NOTE:
// OPTIMIZE:
// TEST:

TODO: is used for regular tasks

FIXME: tells something is broken and should be fixed quickly

BUG: indicates a bug happens in that part of the code

WARN: is a call that something should be changed at some point

HACK: indicates the implementation was done quickly and needs refactoring

NOTE: is just a general note

OPTIMIZE: indicates code needs optimization

TEST: indicates something related to the CMake tests
