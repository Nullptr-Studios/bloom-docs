Detailed Description

Centralized logging system for the engine and game.

The Log class provides a unified interface for handling logging operations within the engine and game. It acts as a wrapper around the spdlog library, offering convenient methods to log messages at various severity levels (e.g., trace, info, warning, error, critical).

This class manages two primary logging consoles:

  • Bloom Console: Used for logging messages related to the engine's internal systems.
  • Game Console: Used for logging messages specific to the game logic and runtime.
